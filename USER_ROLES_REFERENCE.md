# User Roles Reference

This document lists the users created by the seed script and their roles. Actual user IDs will vary based on database state.

## How to Get Current User IDs

Run the following command to see all users with their current IDs:

```bash
npm run list-users
```

Or directly:

```bash
tsx scripts/list-users-by-role.ts
```

## Role Definitions

### LAMS Roles

- **`admin`**: System Administrator - Full access to all features
- **`case_officer`**: Case Officer - Can create/edit SIAs, notifications, awards, possession
- **`legal_officer`**: Legal Officer - Can approve and publish notifications, review objections
- **`finance_officer`**: Finance Officer - Can approve awards, create/approve payments
- **`auditor`**: Auditor - Read-only access to all data and reports
- **`citizen`**: Citizen - Access to citizen portal, can submit feedback and objections

### Legacy Roles (for backward compatibility)

- **`analyst`**: Analyst - Investment management
- **`manager`**: Manager - Approval workflows
- **`committee_member`**: Committee Member - Approval workflows
- **`finance`**: Finance - Same as `finance_officer`

## Role Permissions Summary

| Role | Key Permissions |
|------|----------------|
| `admin` | All permissions |
| `case_officer` | SIA create/edit/publish, notification create/edit, objection resolve, valuation, award, possession |
| `legal_officer` | Notification approve/publish, objection review |
| `finance_officer` | Award approve, payment create/approve |
| `auditor` | View all data, view reports |
| `citizen` | SIA feedback, submit objections, view documents |

## Notes

- User IDs are auto-generated by the database and will vary
- The seed script checks for existing users by username and will not create duplicates
- If a user already exists, the script will use the existing user ID
- To see actual user IDs in your database, run `npm run list-users`
